# @travelhub/types

Shared JSDoc type definitions for resources defined in
`API_SPECIFICATION.md` and `OPENAPI_SCHEMA_REFERENCE.md`, consumed by
both `apps/web` and `apps/api` so a resource shape is documented once,
not twice.

**Sprint 1 status:** scaffolded, empty. Populated incrementally as each
module ships its first real endpoint — see `src/index.js` for the
convention new type files must follow.
